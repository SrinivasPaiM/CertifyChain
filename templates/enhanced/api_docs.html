Zero-Knowledge Proofs

ZK proofs allow refugees to prove eligibility WITHOUT revealing identity:

How It Works:
  1. Refugee generates a commitment (hash) from their eligibility data
  2. ZK circuit proves: "eligibility_score >= min_score" WITHOUT revealing actual score
  3. Service provider only sees: proof is valid/invalid
  4. They CANNOT see: name, ID, address, nationality, or exact eligibility score
Public Signals (what verifier sees):
  • public_signals[0] - Service type (0=healthcare, 1=education, etc.)
  • public_signals[1] - Minimum score required
  • public_signals[2] - Eligible (1) or not (0)
