Overview

CertifyChain SSI provides a privacy-preserving identity system for refugees using:

  • Self-Sovereign Identity (SSI) - Refugees own their identity
  • Zero-Knowledge Proofs (ZK) - Prove eligibility without revealing identity
  • AI Service Matching - Personalized recommendations
  • Blockchain - Immutable verification

System Flow

  1. Admin issues certificate - Verify refugee status
  2. Create DID - Generate decentralized identifier
  3. AI Matching - Get service recommendations
  4. Generate ZK Proof - Prove eligibility without identity
  5. Verify - Service provider validates proof

API Endpoints

POST /zk/proof/

Generate ZK proof for service eligibility

Request: 
{
  "certificate_id": "REF-XXX-1234567890",
  "service_type": "healthcare"
}
Response: 
{
  "success": true,
  "did": "did:ethr:0x...",
  "shareable_proof": { "zk_proof": {...} }
}
POST /eligibility/verify/

Verify ZK proof (for service providers)

Request: 
{
  "proof": {
    "zk_proof": {
      "public_signals": [0, 50, 1]
    }
  }
}
Response: 
{
  "valid": true,
  "message": "Eligibility verified - access granted",
  "privacy_preserved": true
}

Zero-Knowledge Proofs

How ZK Works in This System:
  1. Refugee has a certificate (verified by admin)
  2. System generates a commitment (hash) of their eligibility
  3. ZK circuit proves: "eligibility_score >= min_score" WITHOUT revealing actual score
  4. Service provider only sees: proof is valid/invalid
  5. They CANNOT see: name, ID, address, nationality, or exact eligibility score
Public Signals (what verifier sees):
  • public_signals[0] - Service type (0=healthcare, 1=education, etc.)
  • public_signals[1] - Minimum score required
  • public_signals[2] - Eligible (1) or not (0)
🔧 Setting Up Real ZK Proofs (Windows)

Install Rust (which includes cargo):

Step 1: Install Rust
# Download and run from:
https://rustup.rs

Or use PowerShell:

iwr -useb get.scoop.sh | iex
scoop install rustup
Step 2: After installing Rust
rustup default stable
cargo install circom
npm install -g snarkjs
Step 3: Compile circuits
cd zk-circuits
circom eligibility.circom --r1cs --wasm --sym

# Trusted setup
snarkjs powersoftau new bn128 12 ptau.json
snarkjs powersoftau contribute ptau.json --name="contrib" -v
snarkjs powersoftau export ptau.json pot12_0000.ptau
snarkjs groth16 setup eligibility.r1cs pot12_0000.ptau eligibility_0000.zkey
